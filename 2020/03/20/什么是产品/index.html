什么是产品？

2020年03月20日 1.1k 字 大概 4 分钟
  1. 1. 起因
  2. 2. “产品” 的定义到底是什么？
  3. 3. 总结

起因

为什么会想到 “什么是产品？” 这个问题呢？还要从前几天和同事的一次谈话说起。

谈话具体内容就不说了，总之在谈话的过程中，我发现我对 “产品” 这个概念的理解不是很清晰，不能很清楚地表达出我自己的观点。

因此，这引起了我对 “产品” 这个概念的思考，也因此，才有了这篇文章，来记录对于 “产品” 这个概念的描述。

“产品” 的定义到底是什么？

首先，我找到了百科上对于 “产品” 的定义：

产品是指做为商品提供给市场，被人们使用和消费，并能满足人们某种需求的任何东西，包括有形的物品、无形的服务、组织、观念或它们的组合。

对于这个定义，我并不是很满意。我想要的回答，其实是针对于程序员这个群体的产品的定义；或者更宽泛一点来说，其实我想要的是 IT 行业的产品的定义；又或者更狭义一点，针对于我目前所处的软件开发行业，我想要的回答其实是一些软件相关的产品的定义。

而百科上的回答显然包括了所有行业、硬件、软件等各种产品。

不过，虽然这个回答并不是很符合我的预期，但是回答中的 “包括有形的物品、无形的服务、组织、观念或它们的组合” 这句我还是很认同的。

在我最初的理解中，产品(软件产品)是一个应该能看得见的软件产品，应该包括一些 Web 网站、移动 App 、PC App 等这样的看得见的，有界面的，或者说是位于幕后的 Server 端的软件应用，这样的东西才能被称为产品。

但是，在和同事的谈话过程中，他们的一些观点以及对我目前所在的公司的一些价值观的解释，以及从一些更加专业的角度来对产品的定义所做的一些阐述，让我对我最初的理解有了一些改变。

在同事的观点中，他们提到，产品也可以是对客户的服务，对客户的咨询问题所做出的解决方案，以及一些我所理解的有界面的软件产品。

这让我很认同！

但是，为了更准确地得到 “产品” 这个概念的更精确的表达，或者说是定义，我还是决定查询一番，尤其是在 IT 行业中的一个重要角色——产品经理的角度来对 “产品” 这个概念做出的定义。

我找到的几篇文章：

什么是产品？什么是好的产品？

什么是产品？真的人人都是产品经理吗？

究竟什么是产品经理？

在这几篇文章中，无一例外，作者对 “产品” 这个概念的理解，似乎都是一些 “有界面，看得见” 这样的一些软件产品。除了最后一篇文章所说的 “产品就是载体，是需求的载体” 这句话我觉得还可以外，其实这些文章对于我对 “产品” 这个概念的理解并没有大的帮助。

或者，我也有一个猜想，会不会是 “产品” 这个概念其实在 IT 行业就是这样的理解呢？

其实是我自己想复杂了？

我又找了一些文章，但是，和上面的几篇文章其实大同小异，对于 “产品” 这个概念，并没有非常清晰，非常正式的定义。

我想，对于 “产品” 的概念，也许等以后有幸我做了一名产品经理，可能我会更有体会一些。

最起码，以我目前 Developer 的角色来说，以上文章中的解释及帮助已经够我用了！

或者，等我再工作几年后，可能对于这个概念我会有一些更加深刻的理解！

总结

产品(IT 产品)，包括看得见的有界面的 Web 网站、PC App、移动 App ，看不见的 CLI 、API ，以及一些服务、咨询、解决方案，都可以称为产品。最重要的是，产品是需求的载体，是需求的解决，产品可以作为商品提供给某些客户或市场，来满足它们的需求，被用户所消费，使用，为其带来价值。

产品
